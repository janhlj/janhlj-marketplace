# AI 生成文本的降AI提示词模板

当需要从零生成内容（而非改写）时，使用这些提示词来让生成的文本天然具备人类写作特征。

## 通用降AI生成提示词

### 基础版
```
请以学术论文的风格撰写以下内容。要求：
1. 句子长度要有明显变化，长短句交替使用
2. 避免使用"综上所述"、"首先...其次...最后"、"由此可见"等模板化表达
3. 加入具体的数据、案例或引用
4. 论证要展示思考过程，而非仅陈述结论
5. 适当使用插入语、倒装句等复杂句式
6. 段落长度要参差不齐
7. 术语使用要密集且准确
```

### 进阶版
```
请以[某学科]学者的写作风格撰写以下内容。模仿人类学者的行文特点：
1. 开头不要用宏观背景，从一个具体问题/数据/案例切入
2. 论证中展示思考的"毛边"：自我质疑、修正、补充
3. 至少引用2个真实的学者和他们的具体观点（标注年份）
4. 结论不要完美——指出局限性，提出开放性问题
5. 用词不要总是选最安全的那个，偶尔用出人意料的表达
6. 连接词不要太多，人类写作不会处处加过渡词
7. 至少有一段只有1-2行，用于强调或转折
```

### 深度版（针对最难通过的检测系统）
```
请以以下要求撰写学术内容：

写作风格：
- 句子长度波动大（5字短句与40字长句并存）
- 段落长度不平均（有2行的段，也有10行的段）
- 至少2处使用插入语结构（"——虽然……——"）
- 至少1处使用倒装句
- 至少1处使用省略句
- 不使用以下表达：综上所述、总而言之、首先/其次/最后、由此可见、不仅/而且、值得注意的是、在当今社会

论证风格：
- 从具体案例切入，而非宏观背景
- 展示推理过程，有自我修正和质疑
- 引用真实存在的文献（作者+年份+具体观点）
- 结论包含局限性讨论和开放性问题
- 至少一处引入与主流观点不同的声音

术语使用：
- 在关键论证处密集使用领域专有名词
- 包含该学科内的简称和行话
```

## 按场景分类

### 文献综述
```
请以文献综述的形式撰写关于[主题]的研究进展。要求：
- 按时间线和学术争议两条线索推进
- 指出不同学者之间的矛盾和分歧
- 表达你对这些分歧的个人看法
- 指出当前研究的空白
- 引用格式为（作者，年份）
```

### 实证分析
```
请分析以下[数据/案例/实验结果]。要求：
- 先描述发现，再解读含义
- 提到意外或反直觉的结果
- 讨论可能的替代解释
- 承认方法的局限性
- 用具体的统计数字说话
```

### 理论论述
```
请论述[理论/概念]。要求：
- 追溯概念的学术渊源
- 对比不同学者的定义和理解
- 提出你自己的理解或修正
- 用一个具体例子说明该理论的应用
- 指出理论的边界和局限
```

### 结论部分
```
请撰写论文结论。要求：
- 不要用"综上所述"开头
- 总结核心发现但指出边界条件
- 提出1-2个开放性问题
- 讨论实际意义和政策建议
- 承认研究的不足之处
- 指向未来研究方向
```
