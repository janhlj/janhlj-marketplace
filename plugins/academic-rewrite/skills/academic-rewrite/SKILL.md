# 学术论文降重 & 去AIGC检测

## 核心认知：检测系统的本质

学术检测系统分为两类，应对策略完全不同：

| 检测类型 | 检测什么 | 应对核心 |
|----------|----------|----------|
| **传统查重** | 文字与他人论文的重复度 | 改写表达方式，保留原意 |
| **AIGC检测** | 文本是否具备AI生成特征 | 增加人类写作痕迹，打破AI模式 |

**关键区别**：传统查重看"和别人像不像"，AIGC检测看"像不像机器写的"。

## 知网 AIGC 检测双链路原理（2026版）

```
文本输入 → [链路1: 语义指纹识别] ──┐
                                    ├→ 综合判定 → AIGC%
              [链路2: 写作特征值分析] ──┘
```

- **语义指纹**：提取深层语义模式，与百万学术语料库比对。同义词替换已失效。
- **写作特征值**：分析困惑度、突发性、句式模板、术语密度、论证深度等指标。

## ⚠️ 实战检测数据（2026-04-10 知网真实报告）

以下数据来自一篇医学科普文的真实知网AIGC检测，是优化本 skill 的核心依据：

| 指标 | 原文 | 第一轮改写后 |
|------|------|------------|
| **AIGC 特征值** | 99.9% | **29.7%** |
| **总字符** | ~1500 | 2630 |
| **AI特征字符** | ~1500 | 780 |

### 分段检测结果（关键发现）

| 区域 | AI特征值 | 对应内容 | 结论 |
|------|----------|----------|------|
| **前部20%** | **0.0%** | 标题、作者、具体数据开头的引言 | ✅ 通过 |
| **中部60%** | **16.2%** | 病因分析（基因、环境、生活、感染、炎症、遗传） | ✅ 大部分通过 |
| **后部20%** | **99.8%** | 预防措施+家族史建议 | ❌ 几乎全红 |

### 从实战中总结的"有效"与"无效"

#### 有效的（被知网放行的改写策略）
- ✅ **具体数据开头**：用WHO报告数据（1930万）替代"在日常生活中..."
- ✅ **分子机制描述**：E6/E7蛋白、NF-κB通路、p53、BRCA1 — 全部通过
- ✅ **口语化比喻**："运气不好"、"温床"、"门槛" — 通过
- ✅ **插入语和破折号**：制造句子不规则性 — 通过
- ✅ **具体比例数据**：15%-20%、5%-10%、50%-80% — 通过

#### 无效的（仍被标红的改写策略）
- ❌ **"第一层/第二层/第三层"框架**：从6段对称改为3层后仍被全标 — 框架本身就是AI指纹
- ❌ **建议/指南式文体**："建议收集…"、"必要时可以咨询…" — AI典型推荐模式
- ❌ **"被"字句+正面评价**："被严重低估"、"被证明可以显著降低" — AI高频组合
- ❌ **总结式收尾**："比'一刀切'的体检更有效" — 经典AI收尾模板
- ❌ **"越来越受到重视"**：AI高频四字表达
- ❌ **"不容忽视"**：AI高频成语

### 核心教训

**建议/预防/措施类内容本身就是AIGC检测的高危区域。** 无论怎么重组结构，只要文体是"给建议"，就很难过关。必须从文体层面彻底转换——从"建议式"改为"分析式"。

## 第一步：诊断——定位问题段落

在改写之前，先识别文本中哪些特征会触发检测：

1. **读文本，标记以下问题**：
   - 句式是否过于工整、规律？（低突发性）
   - 用词是否太平淡、太"正确"？（低困惑度）
   - 是否频繁出现模板化表达？（见下方黑名单）
   - 论证是否泛泛而谈，缺少具体案例和数据？
   - 段落结构是否千篇一律（总-分-总）？
   - 术语密度是否偏低？

2. **给出诊断报告**，指出哪些段落AI风险最高、具体原因是什么。

## 第二步：语义重构（对抗语义指纹识别）

这是最核心的一步。**语义指纹关注的是"论证结构"而非"用了哪些词"**。

### 策略

1. **打乱论证顺序**
   - AI 典型模式：观点 → 解释 → 举例 → 总结
   - 改写为：现象/案例切入 → 分析 → 引出观点 → 深入讨论
   - 或者：问题提出 → 反面论证 → 正面回应 → 延伸思考

2. **替换论证素材**
   - 将泛泛的举例换成具体的、有细节的案例
   - 加入数据、年份、人名、事件名等具体信息
   - 引用真实文献（知网会检测引用是否真实存在）

3. **改变因果链条**
   - 如果原文是 "A导致B，B影响C"
   - 可以改为 "C的现象源于B的驱动，而B的根源可追溯至A"
   - 因果方向反转，逻辑等价但语义指纹不同

4. **引入个人视角和批判性思维**
   - 展示思考过程，而非仅陈述结论
   - 提出质疑、对比不同观点

### 不要做

- **不要**只做同义词替换，语义指纹不变
- **不要**只调换语序，远远不够
- **不要**堆砌无意义的长句来"增加复杂度"——检测系统能识别伪复杂

## 第三步：增加困惑度（Perplexity）

### 具体手法

1. **打破高概率搭配**：用不那么"标准"但仍然学术的表达
2. **加入学科特定的非标准表达**：行话、缩写、习惯用语
3. **适度使用非典型句式**：插入语、倒装句、省略句
4. **术语密度提升**：AI 生成的文本在术语使用上"稀释"——害怕出错

## 第四步：增加突发性（Burstiness）

### 具体手法

1. **长短句交替**：AI 句长均匀，人类句长起伏大
2. **段落长度差异化**：不要每段都 5-6 行
3. **打破工整的并列结构**
4. **节奏变化**

## 第五步：消除 AI 模板痕迹

### AI 高频表达黑名单（持续更新，基于实战检测验证）

| AI 模板表达 | 危险等级 | 改写方向 |
|------------|----------|----------|
| "综上所述" / "总而言之" | 🔴 | 直接得出结论 |
| "首先…其次…再次…最后" | 🔴 | 用逻辑推进词 |
| "在当今社会" / "随着科技的发展" | 🔴 | 直接切入具体背景和数据 |
| "被严重低估" | 🔴 实战确认 | "很少有人注意到" / "……的分量不轻" |
| "被证明可以显著" | 🔴 实战确认 | 改为主动描述事实 |
| "越来越受到重视" | 🔴 实战确认 | 直接陈述具体研究进展 |
| "不容忽视" | 🔴 实战确认 | "……同样关键" / "别忘了" |
| "立竿见影" | 🟡 实战确认 | "很快就能看到变化" |
| "具有重要意义" | 🟡 | "这件事的分量不轻" |
| "值得注意的是" | 🟡 | "这里有个细节" / "有趣的是" |
| "建议…"（祈使式） | 🔴 实战确认 | 改为描述性："临床实践中，通常会…" |
| "必要时可以…" | 🔴 实战确认 | 改为陈述事实 |
| "第一层/第二层/第三层" | 🔴 实战确认 | 完全去掉分层框架，用散文式论述 |
| "比…更有效"（收尾） | 🔴 实战确认 | 不用总结式收尾，留白 |

### 消除"AI 味"的检查清单

- [ ] 是否每段都遵循"总-分-总"结构？→ 打破至少 30% 的段落
- [ ] 连接词是否过于丰富和规整？→ 删掉一些
- [ ] 是否所有论点都得到"圆满"论述？→ 留一两个开放性讨论
- [ ] 语气是否始终一致地"学术客观"？→ 加入适度的立场表达
- [ ] 是否缺少具体细节？→ 补充数据、案例、引用
- [ ] 是否包含"建议/应该/可以"等祈使式推荐？→ 改为描述性陈述
- [ ] 是否使用了分层/分级框架？→ 完全去掉，改为散文式
- [ ] 最后一段是否以总结/评价式语句收尾？→ 以事实或未决问题结尾

## 🔴 特殊处理：建议/预防/措施类内容

**这是实战中遇到的最大难题。** 知网检测对"建议式"文体的判定极为敏感——即使内容包含大量具体数据，只要文体是"给建议"，就会被标红。

### 问题本质

AI 大模型天生倾向于生成"建设性建议"，所以"建议/措施/预防"类内容的语义指纹几乎等同于 AI 生成。任何框架（分层、分点、编号）都无法改变这个底层判断。

### 核心策略：文体转换

**从"建议式"转为"分析式"。** 不是说"你应该做什么"，而是说"目前的研究/实践中，人们是怎么做的，效果如何，争议在哪"。

```
❌ 建议式（AI指纹，99.8%被标红）：
  建议收集直系亲属的肿瘤病史信息。
  必要时可以咨询遗传门诊，评估是否需要进行易感基因检测。
  根据风险分层来制定个性化的筛查方案，比"一刀切"的体检更有效。

✅ 分析式（通过知网检测）：
  临床上，家族史的采集正变得越来越细致。
  一些肿瘤中心已经开始常规筛查BRCA1/2等易感基因，
  但是否应该推广到更广泛的人群，学界仍有分歧——
  支持者认为可以提前锁定高危群体，反对者则担心
  过度检测带来的心理负担和医疗资源浪费。
  这个问题目前没有标准答案。
```

### 具体转换手法

| 建议式表达 | 分析式替代 |
|-----------|----------|
| "建议…/应该…" | "在临床实践中，通常的做法是…" |
| "必要时可以…" | "一些专家主张…，但也有不同声音…" |
| "有助于…" | "研究显示，这样做可以…，不过…" |
| "XX是有效的" | "XX的效果在Y研究中得到了验证，但Z研究的结果并不一致" |
| "XX比YY更有效" | "XX和YY各有适用场景，取决于…" |
| "分层编号（第一/第二/第三）" | 去掉编号，用逻辑过渡词自然衔接 |
| 总结式收尾 | 以开放问题、未决争议或具体案例结尾 |

## 第六步：传统查重降重（文字重复率）

1. **改写重复片段**：理解后用自己的话重新表述
2. **引用规范化**：正确标注引用来源
3. **减少直接复制**：定义、法规条文等无法改写的内容使用引用格式

## 工作流

```
1. 接收文本（全文或指定段落）
2. 诊断分析 → 输出问题报告
3. 逐段改写（按上述策略）
4. 输出改写后文本 + 改写说明
5. 可选：提供"改写对比"（原文 vs 改写后的差异标注）
```

### 改写输出格式

对每个改写的段落，提供：

```markdown
### 原文（第X段）
[原文内容]

**诊断**：[指出AI特征/重复问题]

### 改写后
[改写后的内容]

**改写策略**：[说明使用了哪些策略]
```

## 进阶：生成时即降AI

从零生成论文内容时的原则：

1. **开头不要从宏观背景切入**，从具体问题、数据或案例开始
2. **论证要有"毛边"**：不完全对称、不完全工整
3. **加入思考的痕迹**：自我质疑、修正、补充
4. **引用要具体**：有作者名、年份、具体观点
5. **结论不要完美**：指出局限性和未来研究方向
6. **用词的"温度"要有变化**：有些地方精确冷静，有些地方带评价色彩
7. **预防/建议类内容**：用分析式文体替代建议式文体（见上方"特殊处理"一节）

## 重要提醒

- 本技能的目的是帮助作者**更好地表达自己的原创思想**，使其不被检测系统误判
- 改写的前提是**理解内容**——不理解就改，改出来的东西逻辑会断裂
- 最可靠的降AI方法是**真正的人类审校**
- 检测系统在持续升级——理解原理比记住技巧更重要
