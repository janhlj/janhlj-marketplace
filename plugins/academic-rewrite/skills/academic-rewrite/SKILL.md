# 学术论文降重 & 去AIGC检测

## 核心认知：检测系统的本质

学术检测系统分为两类，应对策略完全不同：

| 检测类型 | 检测什么 | 应对核心 |
|----------|----------|----------|
| **传统查重** | 文字与他人论文的重复度 | 改写表达方式，保留原意 |
| **AIGC检测** | 文本是否具备AI生成特征 | 增加人类写作痕迹，打破AI模式 |

**关键区别**：传统查重看"和别人像不像"，AIGC检测看"像不像机器写的"。这是两个完全不同的问题，需要分别处理。

## 知网 AIGC 检测双链路原理（2026版）

理解检测机制才能有效应对：

```
文本输入 → [链路1: 语义指纹识别] ──┐
                                    ├→ 综合判定 → AIGC%
              [链路2: 写作特征值分析] ──┘
```

- **语义指纹**：提取深层语义模式，与百万学术语料库比对。同义词替换已失效——系统看的是"语义骨架"而非字面文字。
- **写作特征值**：分析困惑度、突发性、句式模板、术语密度、论证深度等指标。
- **学科适配**：不同学科有不同的人类写作基准，阈值不同。

## 第一步：诊断——定位问题段落

在改写之前，先识别文本中哪些特征会触发检测：

1. **读文本，标记以下问题**：
   - 句式是否过于工整、规律？（低突发性）
   - 用词是否太平淡、太"正确"？（低困惑度）
   - 是否频繁出现模板化表达？（"综上所述"、"由此可见"、"首先…其次…最后"）
   - 论证是否泛泛而谈，缺少具体案例和数据？
   - 段落结构是否千篇一律（总-分-总）？
   - 术语密度是否偏低？

2. **给出诊断报告**，指出哪些段落AI风险最高、具体原因是什么。

## 第二步：语义重构（对抗语义指纹识别）

这是最核心的一步。**语义指纹关注的是"论证结构"而非"用了哪些词"**。

### 策略

1. **打乱论证顺序**
   - AI 典型模式：观点 → 解释 → 举例 → 总结
   - 改写为：现象/案例切入 → 分析 → 引出观点 → 深入讨论
   - 或者：问题提出 → 反面论证 → 正面回应 → 延伸思考

2. **替换论证素材**
   - 将泛泛的举例换成具体的、有细节的案例
   - 加入数据、年份、人名、事件名等具体信息
   - 引用真实文献（知网会检测引用是否真实存在）

3. **改变因果链条**
   - 如果原文是 "A导致B，B影响C"
   - 可以改为 "C的现象源于B的驱动，而B的根源可追溯至A"
   - 因果方向反转，逻辑等价但语义指纹不同

4. **引入个人视角和批判性思维**
   - 加入 "值得注意的是"、"一个容易被忽视的角度是"、"这与XX的观点形成对照"
   - 展示思考过程，而非仅陈述结论
   - 提出质疑、对比不同观点

### 不要做

- **不要**只做同义词替换（"因此"→"所以"），语义指纹不变
- **不要**只调换语序（"A影响了B"→"B被A影响"），被动变主动稍好但也远远不够
- **不要**堆砌无意义的长句来"增加复杂度"——检测系统能识别伪复杂

## 第三步：增加困惑度（Perplexity）

困惑度衡量用词的"不可预测性"。AI 选择高概率词，人类用词更跳跃。

### 具体手法

1. **打破高概率搭配**
   - AI 倾向： "这一现象表明了问题的严重性"
   - 改写： "这个现象把问题的严重程度摆到了台面上"
   - 用不那么"标准"但仍然学术的表达

2. **加入学科特定的非标准表达**
   - 领域内的行话、缩写、习惯用语
   - 特定学派的术语
   - 行业内的简称

3. **适度使用非典型句式**
   - 插入语： "这个结果——虽然与预期不符——却揭示了更深层的机制"
   - 倒装句： "更为关键的，是样本选择时的偏差"
   - 省略句： "问题不在方法，在前提"

4. **术语密度提升**
   - 在关键论证处密集使用领域专有名词
   - AI 生成的文本往往在术语使用上"稀释"——害怕出错
   - 人类专家的文本术语密度更高

## 第四步：增加突发性（Burstiness）

突发性衡量句子长度和结构的变化程度。AI 文本句子长度均匀，人类写作起伏大。

### 具体手法

1. **长短句交替**
   ```
   AI模式：句1(18字) 句2(20字) 句3(19字) 句4(21字)
   人类模式：句1(35字，复合结构) 句2(6字) 句3(8字) 句4(28字)
   ```

2. **段落长度差异化**
   - 不要每段都 5-6 行
   - 有些段落可以只有 1-2 行（强调、转折）
   - 有些可以长达 10+ 行（深度论述）

3. **打破工整的并列结构**
   - AI 喜欢："第一，…；第二，…；第三，…"
   - 改为："首先是…；除此之外还有一个常被忽略的因素——…；最后"

4. **节奏变化**
   - 论证密集处：短句连用，制造节奏感
   - 解释说明处：长句展开，展示深度
   - 转折处：短促的过渡句

## 第五步：消除 AI 模板痕迹

AI 生成的文本有可识别的模板模式：

### 常见 AI 模板及改写方案

| AI 模板表达 | 改写方向 |
|------------|----------|
| "综上所述" / "总而言之" | 直接得出结论，或用 "回到前面的问题" |
| "首先…其次…再次…最后" | 用逻辑推进词："一个起点是…进而…更进一步…" |
| "在当今社会" / "随着科技的发展" | 直接切入具体背景和数据 |
| "这不仅…而且…"（高频出现） | 偶尔用，但交替使用其他递进结构 |
| "值得我们注意的是" | "这里有个细节" / "有趣的是" |
| 每段开头都是过渡句 | 有些段落直接以事实/数据开头 |
| 每段结尾都是总结句 | 有些段落以问题或悬念结尾 |

### 消除"AI 味"的检查清单

- [ ] 是否每段都遵循"总-分-总"结构？→ 打破至少 30% 的段落
- [ ] 连接词是否过于丰富和规整？→ 删掉一些，人类写作不会处处加连接词
- [ ] 是否所有论点都得到"圆满"论述？→ 留一两个开放性讨论，人类会留白
- [ ] 语气是否始终一致地"学术客观"？→ 加入适度的立场表达和评价
- [ ] 是否缺少具体细节？→ 补充数据、案例、引用

## 第六步：传统查重降重（文字重复率）

这一步针对传统查重系统（CNKI 文字复制比检测）：

1. **改写重复片段**
   - 理解原文意思后，用自己的话重新表述
   - 改变句式结构（主动改被动、肯定改双重否定等）
   - 合并或拆分句子

2. **引用规范化**
   - 正确标注引用来源
   - 直接引用加引号并注明出处
   - 间接引用改写后标注

3. **减少直接复制**
   - 定义、法规条文等无法改写的内容使用引用格式
   - 用自己的话概述他人的研究成果

## 工作流

当收到论文降重任务时，按以下流程执行：

```
1. 接收文本（全文或指定段落）
2. 诊断分析 → 输出问题报告
3. 逐段改写（按上述策略）
4. 输出改写后文本 + 改写说明
5. 可选：提供"改写对比"（原文 vs 改写后的差异标注）
```

### 改写输出格式

对每个改写的段落，提供：

```markdown
### 原文（第X段）
[原文内容]

**诊断**：[指出AI特征/重复问题]

### 改写后
[改写后的内容]

**改写策略**：[说明使用了哪些策略，如"语义重构+增加突发性+消除模板表达"]
```

## 进阶：生成时即降AI

如果用户要求从零生成论文内容（而非改写已有文本），在生成时就应用以下原则：

1. **开头不要从宏观背景切入**，从一个具体问题、数据或案例开始
2. **论证要有"毛边"**：不完全对称、不完全工整
3. **加入思考的痕迹**：自我质疑、修正、补充
4. **引用要具体**：有作者名、年份、具体观点
5. **结论不要完美**：指出局限性和未来研究方向
6. **用词的"温度"要有变化**：有些地方精确冷静，有些地方带评价色彩

## 重要提醒

- 本技能的目的是帮助作者**更好地表达自己的原创思想**，使其不被检测系统误判，而非帮助学术不端
- 改写的前提是**理解内容**——不理解就改，改出来的东西逻辑会断裂
- 最可靠的降AI方法是**真正的人类审校**：AI 改写后再由人通读一遍，修补逻辑断裂和不自然的表达
- 检测系统在持续升级，今天的技巧明天可能部分失效——理解原理比记住技巧更重要
